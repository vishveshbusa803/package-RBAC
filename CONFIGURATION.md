# Configuration Guide

This guide explains the key configuration files and settings needed to run the project properly.

## Environment Variables (.env)

Create a `.env` file in your project root by copying `.env.example`:

```bash
cp .env.example .env
```

### Essential Configurations

#### Application Settings

```env
APP_NAME="Param RBAC"
APP_ENV=local
APP_DEBUG=true
APP_URL=http://localhost:8000

APP_KEY=             # Generated by: php artisan key:generate
```

#### Database Configuration

```env
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=param_rbac
DB_USERNAME=root
DB_PASSWORD=

# Create database if not exists:
# mysql -u root -p
# mysql> CREATE DATABASE param_rbac;
```

#### Authentication Settings

```env
SESSION_DRIVER=cookie
SESSION_LIFETIME=120

CACHE_DRIVER=file
CACHE_STORE=file

QUEUE_CONNECTION=sync

# Two-Factor Authentication
GOOGLE_2FA_ENABLED=true
```

#### Mail Configuration (Optional)

```env
MAIL_MAILER=smtp
MAIL_HOST=smtp.mailtrap.io
MAIL_PORT=465
MAIL_USERNAME=your_username
MAIL_PASSWORD=your_password
MAIL_ENCRYPTION=tls
MAIL_FROM_ADDRESS="noreply@example.com"
MAIL_FROM_NAME="${APP_NAME}"
```

#### API Settings (Optional)

```env
SANCTUM_STATEFUL_DOMAINS=localhost,localhost:3000
```

## Package-Specific Configuration

The `param/rbac` package provides:

### Models Available

All models are in `Param\RBAC\Models`:

- **User**: Main user model with Spatie roles
- **AuthSetting**: Authentication methods configuration
- **PasswordRule**: Password complexity rules
- **RoleMaster**: Role definitions
- **UserTwoFactor**: Two-factor authentication

### Usage in Application

To use package models in your application:

#### Option 1: Use Package Models Directly

```php
use Param\RBAC\Models\User;
use Param\RBAC\Models\AuthSetting;

$user = User::find(1);
$authSettings = AuthSetting::getSetting('GOOGLE_2FA');
```

#### Option 2: Publish and Extend Models

Publish models to your app:

```bash
php artisan vendor:publish --provider="Param\\RBAC\\ProjectSetupServiceProvider" --tag="project-setup-models"
```

Then modify them:

```php
// app/Models/User.php
namespace App\Models;

use Param\RBAC\Models\User as PackageUser;

class User extends PackageUser
{
    // Add your custom methods here
}
```

## Database Migrations

### Automatic Loading

Migrations from the package are automatically loaded and can be run with:

```bash
php artisan migrate
```

### Tables Modified/Created

| Table | Purpose | Type |
|-------|---------|------|
| users | User accounts | Modified* |
| authentication_settings | Auth method configurations | Created |
| password_rules | Password complexity rules | Created |
| user_two_factor | 2FA setup and secrets | Created |
| email_login_otp | Email OTP storage | Created |

**Modified: The users table is created by Laravel, the package modifies it to add custom `address` field.

### Custom Migrations

To create your own migrations:

```bash
php artisan make:migration create_custom_table
```

Place them in `database/migrations/`

## Permissions and Roles (Spatie)

### Creating Roles

```php
// In seeder or migration
use Spatie\Permission\Models\Role;

$role = Role::create(['name' => 'admin']);
```

### Assigning Roles to Users

```php
$user = User::find(1);
$user->assignRole('admin');
$user->hasRole('admin'); // true
```

### Checking Permissions

```php
$user->can('edit articles');
$user->hasPermissionTo('edit articles');
```

## Two-Factor Authentication

### Enabling 2FA for User

```php
$twoFactor = UserTwoFactor::create([
    'user_id' => auth()->id(),
    'secret_key' => 'generated-secret',
    'is_enabled' => true,
    'is_setup_completed' => true,
]);
```

### Verifying 2FA Code

```php
// In controller or custom logic
$code = request()->input('code');
// Verify with Google2FA or custom logic
```

## Troubleshooting Configuration Issues

### "No application encryption key has been specified"

**Solution**:
```bash
php artisan key:generate
```

### "SQLSTATE[HY000] [2002] Connection refused"

**Solution**:
- Check DB_HOST, DB_PORT in .env
- Verify MySQL is running
- Create database if not exists

### "TokenMismatchException"

**Solution**:
- Clear sessions: `php artisan session:clear`
- Regenerate application key
- Ensure session driver is configured

### Migrations Not Running

**Solution**:
- Check database connection in .env
- Manual publish if needed: `php artisan vendor:publish`
- Try: `php artisan migrate --refresh`
